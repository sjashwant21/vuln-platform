# Security Policy

## Supported Versions

We take the security of the VulnAssess Platform seriously. We currently provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| v1.0.0  | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

Please do **NOT** report security vulnerabilities via public GitHub issues or discussions. 

Instead, report them privately using GitHub Security Advisories:
1. Go to the [Security Advisories tab](https://github.com/sjashwant21/vuln-platform/security/advisories).
2. Click on **Report a vulnerability**.
3. Fill out the form with the necessary details.

Alternatively, you can email us directly at: **109244010+sjashwant21@users.noreply.github.com**

### What to Include
To help us triage and resolve the issue quickly, please include:
- A detailed description of the vulnerability.
- Step-by-step instructions to reproduce the issue.
- The potential impact or exploitation scenario.
- A proposed CVSS score (if known).
- Any relevant Proof of Concept (PoC) code or screenshots.

## Response Timeline

We are committed to being responsive to security reports:
- **Acknowledgment:** Within 72 hours of receiving the report.
- **Triage:** Within 7 days, we will confirm the vulnerability and provide an estimated timeline for a patch.
- **Resolution:** We aim to resolve critical issues as rapidly as possible and will keep you updated on our progress.

## Out of Scope

The following items are out of scope for security reports:
- Intentional vulnerabilities created as part of the platform's demonstration or testing features.
- Denial of Service (DoS) or rate limit testing against public, hosted instances.
- Missing security headers or best practices that do not lead directly to an exploitable vulnerability.
- Vulnerabilities requiring unlikely or highly complex user interactions.

## Safe Harbor

We consider activities conducted in good faith to discover and report vulnerabilities to be authorized. We will not initiate legal action against you or ask law enforcement to investigate you if you comply with this policy and make a good faith effort to avoid privacy violations, data destruction, and interruption of our services.
